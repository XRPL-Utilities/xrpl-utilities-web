XR-Pulse

Public-source XRPL event feed

XR-Pulse pulls Ripple, XRP, RLUSD, and XRPL coverage from a curated feed set spanning regulatory press, central banks, and crypto media. Every event captures the XRPL price at ingestion and again four hours later, putting the news and the ledger side by side. No editorial scoring. No source ranking. No social scraping.

Pay $0.10 per query, no API key

XR-Pulse implements x402 v2 with the exact scheme on XRPL mainnet. Verification and settlement are delegated to the t54 XRPL facilitator.

  1. 1. POST /events/recent returns 402 Payment Required with two options: XRP or RLUSD.
  2. 2. Sign an XRPL Payment matching one option, base64-JSON-encode it, send as PAYMENT-SIGNATURE.
  3. 3. Server verifies, retrieves the feed, then settles. Returns 200 with the JSON and a PAYMENT-RESPONSE header.
Swagger UI OpenAPI spec agents.json
# pip install x402-xrpl
from x402_xrpl import X402RequestsSession
from xrpl.wallet import Wallet

session = X402RequestsSession(
    wallet=Wallet.from_seed(seed),
    rpc_url="https://xrplcluster.com",
    payment_requirements_selector="XRP",
)

feed = session.post(
    "https://pulse.xrpl-utilities.io/events/recent",
    json={"limit": 50},
).json()

for e in feed["events"]:
    print(e["title"], e["source_appearances"])

Output shape

What you get back

  • events: array of normalized records, newest-first
  • events[].title, primary_url, brief, published_at (UNIX seconds)
  • events[].source_appearances: which feeds carried the story (e.g. coindesk, sec_press, bank_of_japan)
  • events[].appearance_count: how many sources independently reported it; cross_verified set when 2+ within a 48-hour window
  • events[].correlation: XRPL price at ingestion and at +4h with delta percent. Pure measurement, no causal claim
  • events[].active_utility: signal flags fired against the event (cross-verification, on-chain activity, watchlisted-wallet hits)
  • events[].sentinel_context, target_addresses: any XRPL addresses or Sentinel context the ingestion stage attached
  • row_count, limit, schema_url: response envelope

Full field-level details at /schema. Live manifest at /agents.json.

How it works

Curated feeds, tight Ripple/XRP filter, mechanical correlation

Every entry must mention Ripple, XRP, RLUSD, or XRPL. The institutional sources rarely name Ripple, so they fire infrequently but high-signal when they do. The crypto-press sources are tagged for XRP at the source and carry most of the volume.

When two sources publish the same story (titles overlap within a 48-hour window), we collapse them into one record with a cross-verified flag and the secondary URLs listed alongside.

At ingestion we record the XRPL price. Four hours later we record it again and report the change. Pure measurement, no causal claim.
