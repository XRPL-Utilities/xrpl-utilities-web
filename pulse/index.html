XR-Pulse

Public-source XRPL event feed

XR-Pulse pulls events from four primary feeds, normalizes them into one time-ordered stream, and measures XRPL on-chain price movement in the four-hour window after each event. No editorial scoring. No source ranking. When two sources cover the same story we mark it cross-verified by mechanical title-similarity, not by editorial review.

For developers & AI agents

Pay $0.10 per query, no API key

XR-Pulse implements x402 v2 with the exact scheme on XRPL mainnet. One paid call returns the normalized event log so you do not have to maintain N RSS parsers yourself.

  1. 1. POST /events/recent returns 402 Payment Required with two options: XRP or RLUSD.
  2. 2. Sign an XRPL Payment matching one option, base64-JSON-encode it, send as PAYMENT-SIGNATURE.
  3. 3. Server verifies, retrieves the feed, then settles. Returns 200 with the JSON and a PAYMENT-RESPONSE header.
Swagger UI OpenAPI spec agents.json
# pip install x402-xrpl
from x402_xrpl import X402RequestsSession
from xrpl.wallet import Wallet

session = X402RequestsSession(
    wallet=Wallet.from_seed(seed),
    rpc_url="https://xrplcluster.com",
    payment_requirements_selector="XRP",
)

feed = session.post(
    "https://pulse.xrpl-utilities.io/events/recent",
    json={"limit": 50},
).json()

for e in feed["events"]:
    print(e["title"], e["source_appearances"])

How it works

Four feeds, one stream, mechanical correlation

bis_speeches

Bank for International Settlements RSS for central-bank speeches. Source type: central bank. Captures CBDC, cross-border settlement, and monetary policy framing.

sec_press

SEC press release RSS, filtered post-fetch for entries that mention "ripple" or "xrp" in title or summary. Source type: regulatory press.

cointelegraph_ripple

Cointelegraph's Ripple tag RSS. Source type: crypto news. Source-side topic-filtered, no client-side keyword filter applied.

decrypt_ripple

Decrypt's Ripple category RSS. Source type: crypto news. Source-side topic-filtered, no client-side keyword filter applied.

When two sources publish the same story under different URLs, we cluster them by title similarity. If their normalized title tokens overlap by Jaccard ≥ 0.5 within a 48-hour window, the cluster collapses into one record with cross_verified: true and a cluster_members list so you can verify the merge. The threshold is mechanical, not editorial.

Events older than the four-hour correlation window also carry an optional measurement of XRPL current_price_usd drawn from XR-Telemetry. Every correlation block is watermarked "correlation": "heuristic_only" because temporal proximity is not causation.

Safe harbor & compliance

XR-Pulse is a deterministic event feed, not an editorial product. Source presence is reported as a property of an event, not a verdict on the source. Correlation against XR-Telemetry is a measurement, not a causal claim.

XRPL-Utilities provides public-source data normalization and on-chain measurement. We do not provide financial advice, risk ratings, source credibility assessments, or investment forensics. Users are responsible for ensuring their use of these tools complies with local digital asset and AI regulations, including the California Digital Financial Assets Law and the Colorado AI Act where applicable.
