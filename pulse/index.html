XR-Pulse

Public-source XRPL event feed

XR-Pulse pulls Ripple, XRP, RLUSD, and XRPL coverage from nine feeds spanning regulatory press, central banks, and crypto media. Every event captures the XRPL price at ingestion and again four hours later, putting the news and the ledger side by side. No editorial scoring. No source ranking. No social scraping.

For developers & AI agents

Pay $0.10 per query, no API key

XR-Pulse implements x402 v2 with the exact scheme on XRPL mainnet. One paid call returns the normalized event log so you do not have to maintain N RSS parsers yourself.

  1. 1. POST /events/recent returns 402 Payment Required with two options: XRP or RLUSD.
  2. 2. Sign an XRPL Payment matching one option, base64-JSON-encode it, send as PAYMENT-SIGNATURE.
  3. 3. Server verifies, retrieves the feed, then settles. Returns 200 with the JSON and a PAYMENT-RESPONSE header.
Swagger UI OpenAPI spec agents.json
# pip install x402-xrpl
from x402_xrpl import X402RequestsSession
from xrpl.wallet import Wallet

session = X402RequestsSession(
    wallet=Wallet.from_seed(seed),
    rpc_url="https://xrplcluster.com",
    payment_requirements_selector="XRP",
)

feed = session.post(
    "https://pulse.xrpl-utilities.io/events/recent",
    json={"limit": 50},
).json()

for e in feed["events"]:
    print(e["title"], e["source_appearances"])

How it works

Eight feeds, tight Ripple/XRP filter, mechanical correlation

Every entry must mention Ripple, XRP, RLUSD, or XRPL in the title or summary. The institutional sources rarely name Ripple, so they fire infrequently. When they do, the events are high-signal. The crypto-press sources are tagged for XRP at the source and carry most of the volume.

When two sources publish the same story, we collapse them into a single record by comparing the titles. If the wording overlaps enough within a 48-hour window, the entry gets a cross-verified flag and the secondary URLs are listed alongside, so you can confirm the match yourself.

At ingestion we record the XRPL price. Four hours later we record it again and report the change. Pure measurement, no causal claim.

Safe harbor & compliance

XR-Pulse is a deterministic event feed, not an editorial product. Source presence is reported as a property of an event, not a verdict on the source. Correlation against XR-Telemetry is a measurement, not a causal claim.

XRPL-Utilities provides public-source data normalization and on-chain measurement. We do not provide financial advice, risk ratings, source credibility assessments, or investment forensics.
